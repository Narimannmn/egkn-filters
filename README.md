# ЕГКН Фильтр

Chrome-расширение для удобной работы с участками земли на портале [map.gov4c.kz/egkn](https://map.gov4c.kz/egkn/) (Единый государственный кадастр недвижимости РК).

## Проблема

Штатный интерфейс ЕГКН на `map.gov4c.kz/egkn/` неудобен для системного просмотра участков:

- участки показываются только на карте — нет сводного списка с адресами, площадями и датами;
- нельзя одной операцией загрузить и пролистать **все** участки выбранного слоя по району — карта подгружает данные постранично по мере перемещения;
- нет быстрой фильтрации по статусу (например, «частная собственность», «временное возмездное долгосрочное землепользование» и т.д.);
- ID участков невозможно скопировать одним кликом — приходится открывать карточку и выделять вручную.

Из-за этого первичный анализ заявок по району превращается в долгое ручное кликанье по карте.

## Что делает расширение

Добавляет на страницу `map.gov4c.kz/egkn/*` плавающую кнопку «🔍», которая открывает отдельную панель-диалог поверх карты со следующими возможностями:

- **Авто-подстановка района.** При открытии расширение делает `POST /rest/cabinet/login` и подставляет уже активный на бэке район — пользователю не нужно искать его вручную.
- **Выбор области и района** через каскадный селектор (область → район). Смена района применяется на портале ЕГКН и сбрасывает кэш загруженных слоёв.
- **Все группы слоёв** из ЕГКН (земельные участки, аукционы и т.д.) — без жёсткого whitelist.
- **Универсальный рендер карточек** через адаптер по слою: для каждого типа слоя (например, `reqland`, `free`) свои поля — адрес, площадь, дата, стартовая цена, способ аукциона, категория земель и т.д.
- **Фильтр по статусам** (подслоям) с мультивыбором, счётчиками по каждому статусу и опцией «без статуса».
- **Полная подгрузка всех страниц** выбранного слоя через API ЕГКН с прогресс-баром и счётчиком «загружено N из M». Для больших районов (>2000 участков) автоматически переключается на `pageSize=50`, чтобы не делать тысячи запросов.
- **Пагинация** результатов по 5 карточек на страницу; элементы управления зафиксированы внизу модалки.
- **Копирование ID** участка в буфер обмена одним кликом (с фолбэком на `document.execCommand` для старых браузеров).

Всё работает поверх существующего портала — расширение использует те же публичные REST-эндпоинты, что и сам сайт ЕГКН, и ничего не отправляет наружу.

## Технические детали

- **Manifest V3** content script, инжектируется на `https://map.gov4c.kz/egkn/*`.
- UI монтируется в **Shadow DOM**, чтобы стили сайта и стили расширения не конфликтовали.
- **React 19** + **TanStack Query v5** (`useSuspenseQuery` для мета-запросов, `useQueries` для постраничной загрузки слоя).
- **Suspense + ErrorBoundary** на уровне диалога — состояния загрузки/ошибки декларативно, без ручных тернарников.
- Кэш TanStack Query настроен на `staleTime: Infinity` + `gcTime: Infinity` — повторное открытие района или возврат к уже загруженному слою не вызывает повторных запросов.
- Radix UI (Dialog / Popover / Checkbox).
- Сборка — Vite + `@crxjs/vite-plugin`. CSS разделён по компонентам в `src/styles/` и склеивается через `@import` на этапе сборки, в Shadow DOM прилетает одной строкой.
- **Стриминговая загрузка слоя**: lead-запрос узнаёт `count`, далее остальные страницы запрашиваются волнами по 20 параллельно (`useQueries` + `enabled`), чтобы не перегружать сервер и не упереться в его rate-limit. Abort при смене района — бесплатно через TanStack.

## Использование

1. Соберите расширение:
   ```bash
   pnpm install
   pnpm build
   ```
2. Откройте `chrome://extensions`, включите режим разработчика и нажмите **Загрузить распакованное** → выберите папку `dist/`.
3. Перейдите на `https://map.gov4c.kz/egkn/` — в левом нижнем углу появится синяя круглая кнопка «🔍».
4. Откройте панель: район из вашей сессии подставится автоматически. Выберите группу, слой и статусы — расширение загрузит и покажет все подходящие участки.

## Разработка

```bash
pnpm dev      # vite dev (HMR)
pnpm build    # tsc --noEmit && vite build → dist/
pnpm preview
```

Структура исходников:

```
src/
├── content/              # точка входа content-script (index.tsx) + корневой App.tsx
├── components/           # FAB, диалог, ErrorBoundary, селекторы, список карточек, CopyButton
├── hooks/
│   ├── useLayersMeta / useRegions / useActiveDistrict   # one-shot suspense-запросы
│   ├── useDistrictSelection / useLayerSelection         # state + правила сброса
│   ├── useLayerPages                                    # стриминг страниц волнами
│   └── useSublayerFilter / useStatusCounts              # производные данные
├── lib/
│   ├── api.ts            # REST-клиент /rest/map/layer
│   ├── layersMeta.ts     # /rest/map/layers — группы и подслои
│   ├── districtsMeta.ts  # /rest/map/districts + cabinet/login + PUT /map/district
│   ├── layerAdapter.ts   # схема полей карточки по имени слоя (reqland, free, …)
│   ├── sublayerFilter.ts # чистая фильтрация по статусу
│   ├── statusCounts.ts   # подсчёт элементов по статусу
│   ├── queries.ts        # suspenseQuery() хелпер
│   ├── queryClient.ts    # настроенный QueryClient
│   ├── errors.ts         # getErrorMessage(unknown)
│   └── portalContainer.tsx
├── types/                # типы ответов ЕГКН
└── styles/               # panel.css — точка входа @import, разбит по компонентам
```
